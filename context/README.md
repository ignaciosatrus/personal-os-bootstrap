# context/ — estado vivo del operador

Aquí vive el contexto cambiante. Lo que era cierto ayer puede no serlo
hoy. Por eso este directorio se lee al inicio de cada sesión.

**Tras el bootstrap (lo que escribirás en sesión 1):**

- `entity.md` — quién eres. Rol, responsabilidades, contexto vital.
  Se escribe en primera persona.
- `now.md` — estado vivo. Qué tienes en la cabeza ahora mismo.
  Se actualiza con frecuencia, idealmente al cierre del día.
- `next-steps.md` — prioridades pendientes. Lista corta, jerarquizada.

**Patrones que pueden aparecer después:**

- Subcarpetas por cliente / proyecto / tema (si tu `filesystem.md`
  lo justifica).
- `learnings/` — captura de conocimiento reusable.
- `meetings/` — transcripciones procesadas.

No los crees por anticipación. Aparecen cuando los necesitas.
